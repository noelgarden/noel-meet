# ノエルミート — 拠点の窓 🪟

複数の実店舗（拠点）をカメラ・音声で **24時間つなぎっぱなし** にする社内アプリです。
相手のオフィスを「窓」に見立て、常に映像が流れ続けることで同じ空間にいる感覚をつくります。

- サーバーレス P2P（PeerJS / WebRTC）。バックエンドなし、静的ファイルのみで動作
- ログイン・URL発行・毎回の入室操作は不要。合言葉と拠点番号を入れるだけ
- 6秒ごとの自動再接続ループで、端末が再起動しても人手なしで復旧
- 画面共有対応（共有中はカメラ映像の代わりに画面が全拠点へ流れる）
- 録画対応（全拠点の映像＋音声を1本の動画に合成。Google ドライブの指定フォルダへ自動保存）
- **運用費用ゼロ**：GitHub Pages・PeerJS クラウド・STUN/TURN すべて無料枠で動作

## デプロイ手順（GitHub Pages）

カメラ利用のため **https が必須** です。GitHub Pages を使うのが最も簡単です。

1. GitHub で新しいリポジトリを作成（例：`noel-meet`。Public / Private どちらでも可。Private の場合は GitHub Pages が有料プランで必要）
2. このフォルダの `index.html` と `README.md` をリポジトリに push

   ```bash
   git init
   git add index.html peerjs.min.js README.md
   git commit -m "ノエルミート 初回コミット"
   git branch -M main
   git remote add origin https://github.com/<ユーザー名>/noel-meet.git
   git push -u origin main
   ```

3. リポジトリの **Settings → Pages** を開く
4. 「Build and deployment」の Source を **Deploy from a branch**、Branch を **main / (root)** にして Save
5. 数分後に `https://<ユーザー名>.github.io/noel-meet/` で公開される

## 各拠点でのセットアップ

1. 各拠点の常設端末（ミニPC / タブレット）の Chrome または Edge で公開URLを開く
2. セットアップ画面で入力：
   - **合言葉** … 全拠点で同じもの（例：`noel2026`）。これが漏れると誰でも参加できるので複雑なものを推奨
   - **拠点の数** … 接続する拠点の総数（2〜5）
   - **この端末の拠点** … 拠点ごとに重複しない番号を割り当てる（本社=1、○○店=2 …）
   - **表示名** … 例「一宮本社」
3. 「この拠点でつなぐ」を押し、カメラ・マイクの許可ダイアログで **許可** を選ぶ
4. 全拠点で起動すれば自動的に相互接続される（順番は問わない）

設定は端末に保存されるため、再起動後も同じ内容が復元されます。

## 画面共有

下部の 🖥️ ボタンを押すと、共有する画面（画面全体 / ウィンドウ / タブ）の選択ダイアログが出ます。
共有中は **カメラ映像の代わりに** その画面が全拠点へ流れます。もう一度 🖥️ を押すか、ブラウザの「共有を停止」でカメラ映像に戻ります。

- 画面共有ができるのは PC 版 Chrome / Edge です（Android タブレット等は非対応。受信は可能）

## 録画

下部の ⏺ ボタンで録画を開始・停止します。**全拠点の映像がグリッド合成され、全拠点＋自分の音声もミックスして** 1本の WebM 動画になります。録画中は上部バーに「● REC 経過時間」が表示されます。

保存先は2通り：

| 設定 | 保存先 |
|---|---|
| Google ドライブ設定済み | 指定フォルダ（未指定ならマイドライブ直下）へ自動アップロード。失敗時は端末へダウンロード |
| 未設定 | この端末へダウンロード保存 |

ファイル名は `ノエルミート録画_2026-07-04_1030_一宮本社.webm` の形式です。

### Google ドライブ保存のセットアップ（初回のみ・約10分）

ブラウザから直接ドライブへアップロードするため、Google の「OAuth クライアントID」が必要です。

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、プロジェクトを作成（例：`noel-meet`）
2. 「APIとサービス → ライブラリ」で **Google Drive API** を検索して有効化
3. 「APIとサービス → OAuth 同意画面」を設定
   - Google Workspace を使っている場合：User Type = **内部**（社内アカウントのみ許可され安全）
   - 個人 Gmail の場合：User Type = **外部** にし、「テストユーザー」に録画で使う Google アカウントを追加
4. 「APIとサービス → 認証情報 → 認証情報を作成 → OAuth クライアントID」
   - アプリケーションの種類：**ウェブ アプリケーション**
   - 承認済みの JavaScript 生成元に公開URLのオリジンを追加：`https://<ユーザー名>.github.io`
5. 発行された `xxxx.apps.googleusercontent.com` を、アプリのセットアップ画面「録画の保存先」欄に貼り付け
6. 保存先フォルダは、ドライブでフォルダを開いたときの **URL をそのまま貼り付け** ればOK（IDだけでも可）

権限は `drive.file` スコープ（このアプリが作ったファイルにしか触れない最小権限）です。初回録画時に Google の許可ポップアップが出ます。

### 録画の注意

- 録画データは停止までブラウザのメモリに貯まります。長時間の連続録画は避け、必要な打ち合わせの間だけ録画してください（目安：1時間以内）
- 相手拠点には録画中であることが自動通知されません。**録画開始時は口頭で一声かける運用にしてください**
- 保存先フォルダには、録画する Google アカウントの編集権限が必要です

## 費用について

**月額費用はかかりません。** 全構成が無料サービスで完結します。

| 構成要素 | サービス | 費用 |
|---|---|---|
| ホスティング | GitHub Pages | 無料（Public リポジトリ） |
| シグナリング | PeerJS 公開クラウド | 無料 |
| NAT越え（STUN） | Google 公開 STUN | 無料 |
| NAT越え（TURN） | openrelay.metered.ca | 無料 |
| 映像・音声の通信 | 拠点間の P2P 直接通信 | 各拠点の既存ネット回線のみ |

映像はサーバーを経由せず拠点間で直接やり取りするため、従量課金が発生する場所がありません。
将来お金がかかる可能性があるのは次の2点だけで、いずれも「問題が起きたら」で構いません：

- 社内ネットワークが厳しく P2P が通らない場合 → 有料 TURN（Metered 等、月数ドル〜）
- PeerJS 公開クラウドの不調が続く場合 → 自前 PeerServer（VPS 月数百円）

## セキュリティ

| 対策 | 内容 |
|---|---|
| 映像・音声の暗号化 | WebRTC 標準の DTLS-SRTP により常時エンドツーエンド暗号化。TURN サーバー経由時も中継サーバーは復号できない |
| 接続IDの推測不能化 | Peer ID は合言葉の SHA-256 ハッシュから導出。合言葉を知らない限り ID 自体を推測できない |
| 着信認証 | 着信時に合言葉由来の認証トークンを検証し、一致しない接続は自動拒否（「認証されていない接続を拒否しました」と表示） |
| 合言葉の強度 | 8文字以上を必須化。🎲 生成ボタンで推測されにくいランダムな合言葉を作成可能 |
| 覗き見対策 | 画面上部の合言葉表示は先頭2文字以外をマスク |
| サプライチェーン | PeerJS ライブラリは CDN でなく自前ホスト（`peerjs.min.js`）。CSP で外部スクリプトの読み込みを禁止 |

### 残るリスクと運用ルール

- **合言葉がすべての鍵**です。社外に漏らさないこと。漏れた疑いがあれば全拠点で合言葉を変更する（🎲 生成 → 全端末で入力し直すだけ）
- 合言葉は端末内（localStorage）に保存されるため、**常設端末自体の物理管理**（持ち出し・盗難対策、OSログインパスワード）はしてください
- シグナリング（接続のとりもち）は PeerJS 公開クラウドを経由しますが、流れるのは接続情報のみで映像・音声は流れません

## 運用上の注意

- **スピーカーは各拠点1台に**。1部屋で複数のスピーカーが鳴るとハウリングします
- エコーキャンセラー付きの会議用スピーカーマイク（例：Anker PowerConf）を推奨
- 画面スリープは Wake Lock API で防止しますが、OS側の電源設定でも「スリープしない」にしておくと確実です
- ブラウザは Chrome / Edge を想定

## 既知の制約

| 項目 | 内容 |
|---|---|
| TURN | 無料の openrelay を使用。帯域・安定性に限界あり。厳格な社内ネットワークでは有料TURN（Metered / Twilio / 自前 coturn）へ差し替えを推奨 |
| 認証 | 合言葉ベース（ハッシュID＋着信トークン検証）。合言葉を知る人は誰でも参加できるため、合言葉の管理が要 |
| シグナリング | PeerJS 公開クラウドに依存。落ちると新規接続不可（確立済みの通話は継続）。本格運用時は PeerServer の自前ホストを検討 |
| 拠点増減 | 起動時に手動選択。拠点を増やす場合は全端末で「拠点の数」を揃え直す |

## 技術構成

- HTML / CSS / Vanilla JavaScript の `index.html`（ビルド不要）＋ 自前ホストの [PeerJS 1.5.4](https://peerjs.com/)
- WebRTC フルメッシュ P2P。決定的 Peer ID `noelwindow-<合言葉のSHA-256先頭20桁>-<拠点番号>` により、ルームサーバーなしで相互発見
- 発信は「自分より番号が大きい拠点のみ」・着信は全応答（二重接続の防止）
